ICGC and Amazon have partnered to offer low-cost and scalable computing solutions for analysing ICGC data on AWS EC2. Raw read data, genome alignments, and associated metadata produced by the ICGC PanCancer Analysis of Whole Genomes (PCAWG) study in addition to data submitted by ICGC projects to the European Genome Archive (EGA) is periodically gathered, curated and incrementally uploaded to Amazon S3 for the benefit of the cancer research community. An up-to-date summary of the repository's data content is provided on the right. Our user guide provides step-by-step instructions on how to access and analyse ICGC data on AWS. This data and associated search and access tools is managed by the ICGC Data Coordination Center (DCC). Read-level data is in the controlled tier, and you will need to obtain approval from the ICGC Data Access Compliance Office (DACO) in order to obtain the decryption keys necessary to access this data.

The data produced by the PCAWG project, including somatic mutations, is subject to a publication embargo. Please see the PCAWG publication and embargo policy for the embargo terms.

PCAWG Workflows

The PCAWG project has created standardized, whole genome workflows for alignment using BWA-Mem and variant calling using best practice processes from the Sanger Institute, the Broad Institute, and DKFZ / EMBL. These are made available to the research community via a "launcher" AMI that allows users to analyze their data using the same workflows. Currently supporting the BWA alignment workflow, this system makes it easy for users to load their data into AWS S3, launch many VMs simultaneously to analyze data in parallel, and write the resulting alignments back to S3. Over time, the full range of PanCancer workflows will be made available.
